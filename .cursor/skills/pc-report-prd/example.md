# 样例 PRD 索引

写新报表 PRD 时，按页面复杂度选用参照样例。

---

## 选用指南

| 报表类型 | 参照样例 | Tab 结构 | 典型按钮 |
|---------|---------|---------|---------|
| **单 Tab 查询报表** | 按本 skill 模板裁剪，仅保留一个 Tab | 查询 + 导出 + 字段表（可无定时任务） | 导出 |
| **明细 + 汇总下钻** | 推荐补贴报表（人员汇总部分） | 明细 → 人员汇总 | 导出 |
| **多层汇总 + 终版导入** | 推荐补贴报表（完整版） | 明细 → 人员汇总 → 分拨汇总 + 终版 Tab | 导出、下载模版、导入 |

## 标杆样例：推荐补贴报表

**完整 PRD**：`/Users/humianfei/Downloads/推荐补贴报表.md`

**定位**：最典型的 PC 端核算报表 — 定时跑数、复杂字段计算、多层汇总下钻、终版人工导入覆盖。

### 文档要点

| 章节 | 内容 |
|------|------|
| 概述 | V1.0.0，胡绵飞，修订说明「新增」 |
| 要解决的问题 | 人工线下核算 + 手动导入薪资，效率低、易错、难追溯 |
| 3.1 产品方案 | 方案配置、自动核算、薪资同步（方案层，非字段细节） |
| 菜单路径 | 数智分拨 - 货量提成 - 薪资线上化 - 特殊货量报表后 |
| 权限 | 菜单权限 + 数据权限 |

### Tab 与 Skill 模板对照

| Tab 名 | Skill 模板类型 | 要点 |
|--------|--------------|------|
| **补贴明细** | 明细 Tab | 查询规则视角切换；定时任务（日/月）；前置说明（纳入口径）；四列字段表（新员工/推荐人嵌套）；方案编号优先级匹配；金额计算与为零条件 |
| **人员汇总** | 人员/对象汇总 Tab | 由明细按月份/分拨/工号等分组；金额列分类型加总；双击下钻至明细（按角色视角） |
| **分拨汇总** | 组织/分拨汇总 Tab | 由人员汇总再按分拨分组；「非分拨人员推荐」特殊行；双击下钻至人员汇总 |
| **人员汇总终版** | 终版/调整 Tab | 查询条件与人员汇总高度相似（用继承写法）；导入三层校验；按补贴月份整月覆盖；创建人/时间 |

### 层级关系

```
补贴明细（源数据 + 计算）
    ↓ 分组汇总
人员汇总（按人）
    ↓ 分组汇总
分拨汇总（按分拨）

人员汇总终版（独立导入层，不自动从下钻生成）
```

### 与 Skill 默认的差异

| 项 | 推荐补贴报表 | Skill 默认 |
|----|------------|-----------|
| Tab 数量 | 4 个 | 按实际裁剪，至少 1 个 |
| 查询规则 | 有（新员工/推荐人视角） | 有则写，无则省略 |
| 定时任务 | 有（日 + 月） | 明细 Tab 有则写 |
| 字段表列数 | 平铺三列；嵌套四列 | 无二级字段用 `字段\|说明\|补充说明`；有分组嵌套用一级/二级字段 |
| 按钮 | 导出为主；终版 Tab 有导入 | 同左 |
| 变更记录 | 无 | 报表默认无 |

## 写新报表 PRD 时的选用建议

| 新报表特征 | 参照 |
|-----------|------|
| 仅查询 + 导出，无跑数任务 | 单 Tab 模板，省略定时任务与前置说明 |
| 有后台定时核算，字段含计算公式 | 推荐补贴报表 · 补贴明细 Tab |
| 明细 + 按人/单汇总 + 双击看下钻 | 推荐补贴报表 · 人员汇总 Tab |
| 再按组织/分拨汇总 | 推荐补贴报表 · 分拨汇总 Tab |
| 需人工导入终版数据覆盖系统计算 | 推荐补贴报表 · 人员汇总终版 Tab |
| 多 Tab 查询条件几乎相同 | 继承写法，只写差异点 |

---

## 注意事项

- 业务计算逻辑（方案匹配、金额公式、为零条件）以用户提供的**讨论结论 / 已确认原型**为准；skill 只约束结构与写法，**不编造**公司内部规则。
- 不确定的枚举、岗位清单、方案表名等标「待确认」。
- 章节 1～4 写法与维护页 PRD 一致，详见 [pc-maintain-prd](../pc-maintain-prd/SKILL.md)。
