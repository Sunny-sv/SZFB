# 版本增量 PRD 写作规范

## 何时使用

- 写 **V1.1.0 / V2.0.1** 等相对上一版的增量 PRD
- 用户说「只写改了什么」「看不懂，重写」「不要引用前面章节」
- 计算逻辑、匹配规则、配置改版等**规则类**增量文档
- 已有 V1.0.0 / V2.0.0 基线，本版只交付变更部分

**不替代** pc-maintain-prd / pc-report-prd：新建维护页、报表页仍用对应 skill 的全量结构；本 skill 管**版本增量**写法。

## 写法风格

必须同时遵守 [pm-prd-writing-style](../pm-prd-writing-style/SKILL.md)：只写有用内容，改动按查询条件、按钮操作、数据列表等模块归类；表格长内容拆有序列表，展示规则等非字段内容放表格最后说明行；语言用日常产品经理口吻。

## 核心原则

| 原则 | 要求 |
|---|---|
| **只写改动** | 正文只描述本版相对上一版的新增、删除、调整；未改内容不单独成节、不展开 |
| **禁止序号互引** | 不写「见 4.3」「进入 4.5～4.6」「按 5.2 取值表」「与 V2.0.0 一致（见第四章）」；需要的内容**当场写全** |
| **禁止符号化** | 不用 D、v(d)、∀、∈ 等数学写法；区间写「下限 ≤ 实际值 < 上限」 |
| **业务可读** | 读者是研发、测试、运营；用「放弃本方案，继续下一层」而非「本方案视为不匹配，continue 4.3」 |
| **自洽可审** | 单读本文档能评审本版交付范围，不必来回翻基线 PRD |
| **不重复维护侧内容** | 维护页已写清的字段枚举、页面限制、导入规则，不在计算逻辑里重复；计算文档只写运行时真正会用到的匹配、取值、命中、兜底、落库 |

## 禁止写法 vs 推荐写法

| 禁止 | 推荐 |
|---|---|
| 见 4.3 方案层级 | 四层顺序：特殊维护-分拨 → 特殊维护-全国 → 统一维护-分拨 → 统一维护-全国 |
| 进入 4.5～4.6 计算 | 按岗位取自有/外包调整比例，再按原百分比重量公式计算 |
| 按 5.2 取值表 | 按下文「区间维度与扫描取值」表 |
| 与 V2.0.0 一致，略 | 不写；没列出的内容默认未改 |
| 本章替代 V2.0.0 中 XXX 段落 | 放回对应模块，直接写「删除」「新增」「调整」 |
| `[800, 99999.99)` | 800～99999.99（正文说明前闭后开即可） |

## 推荐文档结构

增量 PRD 按以下顺序输出（可按场景增减章节，**不**复制基线全文）：

```
一、概述（版本表 + 一句话：本文仅描述 Vx.y.z 相对上一版的变更）
二、本版改动范围
    2.1 本版改动概述（编号列表，3～8 条）
三～N、变更规则正文（按查询条件/按钮操作/数据列表/规则主题分章，每章只写改动）
    例：区间维度与扫描取值 / 匹配条件数量 / 配置行命中规则 …
验收要点
关联文档
```

**不写**「V1.0.0 全文」「V2.0.0 保留部分」「本版没改什么」等基线复述。

## 规则类章节写法

### 改动范围（第二章）

**改了什么**：每条一句话点明「删/增/改」+ 业务效果。

删除、新增、调整要放回所属模块，不单独堆「删除项」总表。

计算逻辑类增量只保留运行时必要规则。维护页已经禁止维护、已经列过字段枚举或评审时已确认的页面规则，不要在计算逻辑里再写一遍。

### 核心规则（如配置行命中）

用 **步骤 + 结果表 + 业务示例**，不用伪代码符号：

1. **第一步：……**（写清输入、动作）
2. **第二步：……**
3. …

| 情况 | 处理 |
|---|---|
| 命中 1 行 | … |
| 0 行命中 | … |

> **举例**：具体数字 + 命中/未命中各一条。

### 验收要点

只验收**本版变更**；回归项可保留 1 条「与改版前一致」。

## 写作前检查

- [ ] 是否只写本版增量，未把基线 PRD 又抄一遍？
- [ ] 全文是否无「见 X.X」「按第 X 章」？
- [ ] 规则章节是否单读可懂，无需翻其他文档？
- [ ] 是否无 D、v(d)、∀ 等符号？
- [ ] 删除、新增、调整是否已放回对应模块？
- [ ] 是否删掉了维护侧已说明、计算侧不需要重复的内容？

## 交付前必审

写完增量/规则 PRD 后，**先启用** [pm-review](../pm-review/SKILL.md) 做逻辑/结构/写法审查（重点查互引、只写改动、规则自洽），输出「待你确认」清单后再结束本轮（用户明确说「先别审」除外）。默认只提问题，不擅自改文档。
