# PC 端原型（HTML 线框）

## 何时使用

- 用户要 **画原型 / PC 端页面 / 查询页 / 维护页 / 列表页 / 配置页界面**
- 处于 pm-core 工作流的 **原型阶段**（讨论已对齐方向，或用户明确要求出界面）
- 用户说「基于 PC 框架」「按通用后台布局画」

**不启用：**
- 用户只要讨论方案、不写界面
- 用户要写 PRD → 改用 `pc-maintain-prd` skill（须原型已确认）
- 用户要 Canvas、Figma、高保真视觉稿

## 母版（唯一布局基准）

**路径：** `docs/_共用/PC端通用框架.html`

**工作方式：**
1. **复制母版** → 放到对应项目目录，命名为 `docs/{项目名}/{中文页面名}.html`
2. **只改占位内容**（菜单名、标签页名、筛选字段、按钮文案、表格列名、行数）
3. **禁止从零发明新布局**；禁止改动顶栏 + 侧栏 + 四模块骨架结构

布局细节对照见 [reference.md](reference.md)。
筛选/表单控件命名与 PRD 对齐时，对照 [pm-component-catalog](../pm-component-catalog/SKILL.md)（线框阶段仍用母版 HTML 控件形态，label 用标准组件名便于后续写 PRD）。

## 交付物

| 项 | 要求 |
|----|------|
| 格式 | 单文件 HTML（内联 CSS + 少量演示 JS） |
| 路径 | `docs/{项目名}/{中文页面名}.html`（与同项目 PRD 同目录） |
| 打开方式 | 浏览器直接打开，无需构建/后端 |
| 保真度 | 低 fidelity 线框；Ant Design 企业后台风格 |
| 依赖 | 不引 CDN、不接 API、不写后端逻辑 |
| **像真页面** | 原型就是实际业务页长什么样；**禁止**在页面上放「原型说明」、黄条注释、IT/PRD 备注、演示专用按钮文案 |

## 禁止页内说明（硬约束）

原型给业务/研发评审用，应看起来像即将上线的页面：

| 禁止 | 正确做法 |
|------|----------|
| 页面顶部/卡片内「原型说明：……」黄条、灰条、虚线备注框 | 说明写在对话回复或 PRD，不写在 HTML 页面上 |
| 「演示用」「仅示意」「一线不可见」「点某某按钮可看」等 meta 文案 | 用真实按钮、真实提示语、真实列表态表达 |
| 为讲解临时加的操作按钮（如「打开××（总额超限）」） | 走真实入口：点行、点编辑、点分配等现网路径；多状态用多行样例数据切换 |
| 把「只存表、列表不展示」的字段画成页面可见区并加注释 | 页面不画该字段；规则写 PRD |

异常提示（如超限红字）属于**真实页面会有的反馈**，可以画；但不要再加一层「这是演示」的说明。

## 布局硬约束

### 全局结构（固定）

```
┌──────────┬────────────────────────────────────────────┐
│ 品牌区    │ 蓝色顶栏（折叠 + 首页 + 页面标签 + 工具区）   │
│ 深色      │                                            │
├──────────┼────────────────────────────────────────────┤
│ 侧边栏    │ 主内容区（四模块，白底 Card）                 │
│ 可折叠    │                                            │
└──────────┴────────────────────────────────────────────┘
```

1. **顶栏整行** — 左：深色品牌区（文案「数智分拨 | DLC」）；右：蓝色顶栏（`#0659d9`），含折叠按钮、首页、页面标签、右侧工具/用户占位
2. **左侧** — 竖向侧边菜单（无独立 Logo 区；折叠由顶栏汉堡按钮控制）
3. **右侧** — 主内容展示区（浅灰底 + 白底 Card）

### 主内容区四模块（自上而下，顺序不可变）

| 序号 | 模块 | 硬约束 |
|------|------|--------|
| 1 | 筛选条件区 | 默认 **一行 3 个**筛选组件（下拉 / 输入 / 日期）；名称与控件 **左右布局**；最右侧固定 **重置 + 查询**，与筛选区 **最后一行底对齐** |
| 2 | 标签切换区 | 横向 Tab 条（纯框架，可切换 active 样式） |
| 3 | 操作按钮区 | 横向功能按钮组 |
| 4 | 数据列表区 | 标准表格：**首列复选框**；**分页固定贴在内容卡片底部**（数据再少也贴底；表体多时在表格区内滚动）；表头与单元格文字默认 **居中对齐** |

### 筛选区扩展规则

- 每个 `.filter-item`：**左** `.filter-label`（右对齐固定宽）+ **右** `.filter-control-wrap`（控件）
- 用户明确要求多行筛选时：每行仍 **3 列均匀排列**，查询/重置 **只在最后一行右侧**（`.filter-area` 用 `align-items: flex-end`）
- 未说明行数时：保持 **单行 3 组件**

### 弹框 / 表单字段布局（硬约束）

- 字段名与输入控件必须 **左右排列**：**左** label（右对齐）+ **右** 控件；**禁止** label 在上、控件在下
- 与筛选区同一原则；多列表单可用 grid 排多列，但每个字段单元内部仍是左右布局
- 长 label（如「单人单月金额限制」）加宽 label 栏，不要改成上下叠放

### 样式约定（与母版一致）

| 用途 | 色值 |
|------|------|
| 顶栏蓝 | `#0659d9` |
| 主色（按钮/菜单选中等） | `#1890ff` |
| 侧边栏 / 品牌区 | `#001529` |
| 内容背景 | `#f0f2f5` |
| 卡片 | `#ffffff` |
| 边框 | `#e8e8e8` |

## 占位与命名规则

| 状态 | 写法 |
|------|------|
| 未确认字段 | 「筛选条件」「下拉框」「输入框」「列 N」「操作按钮 N」「标签 N」「菜单项 N」 |
| 讨论已对齐 | 填真实字段名 / 按钮名 / 列名 |
| 不确定 | 保留通用占位，不编造业务规则 |

## 最小交互（演示用，非业务）

保留母版中的演示交互即可：
- 侧边栏折叠/展开（顶栏汉堡，toggle `#layout.sidebar-collapsed`）
- 顶部标签 / 内容 Tab 切换 active
- 表头全选联动行 checkbox

不实现：查询、重置、分页跳转、菜单路由、表单校验。

## 与 pc-maintain-prd 衔接

```
讨论 → [pc-prototype] 出 HTML 原型 → 评审确认 → [pc-maintain-prd] 写 PRD
```

- 原型阶段：**不出 PRD 长文**
- PRD 阶段：**不擅自改已确认的原型交互**；若需改布局，回到原型阶段

## 常见变更速查

| 用户需求 | 改哪里 |
|----------|--------|
| 换筛选字段 | `.filter-grid` 内 `.filter-item`：`.filter-label` + `.filter-control-wrap` + 控件 |
| 多加筛选条件 | 在 `.filter-grid` 内追加 `.filter-item`，保持 3 列自动换行 |
| 换 Tab 名/数量 | `.content-tabs` |
| 加/减按钮 | `.action-bar` |
| 换表格列 | `<thead>` + `<tbody>` 列数对齐 |
| 换菜单/页面标签 | `.sidebar-menu` / `.page-tabs` |
| 不要内容 Tab | 与用户确认；极少数页面可去掉模块 2，须说明原因 |
| 不要复选框 | 与用户确认；维护页通常保留 |

## 产出前自检

- [ ] 从 `docs/_共用/PC端通用框架.html` 复制，非从零搭建；文件名为中文，落在对应项目目录
- [ ] 全局结构完整：顶栏（品牌区 + 蓝底标签栏）+ 侧边栏 + 主内容
- [ ] 主内容四模块顺序正确：筛选 → Tab → 按钮 → 表格+分页
- [ ] 分页贴在内容卡片底部（非紧跟表体后悬空）
- [ ] 筛选区：名称左+控件右；一行 3 组件 + 右侧重置/查询且与最后一行底对齐
- [ ] 弹框/表单字段：名称左+控件右（禁止上下叠放）
- [ ] 表格文字默认居中（含表头与数据单元格）
- [ ] 表格首列为 checkbox，底部有分页占位
- [ ] 单文件 HTML，无 CDN / 无后端
- [ ] **页面上看不到「原型说明」/演示备注/临时演示按钮**；呈现的是实际业务页
- [ ] 未确认信息用通用占位，不编造业务规则
- [ ] 未混用 PRD 输出

## 交付前必审

原型 HTML 写完后，**先启用** [pm-review](../pm-review/SKILL.md) 做布局/交互审查（有对应 PRD 时一并做字段对齐），输出「待你确认」清单后再结束本轮（用户明确说「先别审」除外）。默认只提问题，不擅自改文档。
