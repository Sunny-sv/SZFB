# 通用组件目录

对照样例：`docs/百分比重量/货量重量配置.md`。写 PRD 时**点名组件 + 只写本页差异**；下列「默认能力」不必在说明列重复。

---

## 1. 基础控件

### 单选下拉框

| 项 | 约定 |
|----|------|
| 标准名 | 单选下拉框 |
| 默认能力（不写） | 点开选一项、可清空（若业务允许） |
| 说明列必写 | 枚举项、默认值、查询逻辑（查询区）；弹框写必填/置灰 |
| 常见模式 | 全部/是/否；全部/全国/分拨；全部/下调/上调 |

**查询条件范例：**

```markdown
|轻实重调整|单选下拉框|1. 枚举项：全部、是、否。<br>2. 默认值：**全部。**<br>3. 查询逻辑：全部含是与否；是/否分别匹配对应字段值。|
```

**弹框范例：**

```markdown
**控件：**单选下拉框；**枚举项：**分拨、全国。**默认值：**空；必填；
```

---

### 多选下拉框

| 项 | 约定 |
|----|------|
| 标准名 | 多选下拉框 |
| 默认能力（不写） | 多选、勾选展示 |
| 说明列必写 | 枚举项或数据源、默认值（常为空=不参与过滤）、查询逻辑（包含 / 有交集） |

**范例：**

```markdown
|操作类型|多选下拉框|1. 枚举值：卸车、移动、装车。<br>2. 默认值：**空，该查询条件不生效、不参与数据过滤。**<br>3. 查询逻辑：选中内容包含操作类型字段内容的数据。|
```

任务类型等多值字段常用「有交集」：

```markdown
|任务类型|多选下拉框|1. 枚举值：分拨、服务站、集配站、网点、智运分拨。<br>2. 默认值：**空，该查询条件不生效、不参与数据过滤。**<br>3. 查询逻辑：选中内容与任务类型字段内容有交集的数据。|
```

---

### 文本输入框

| 项 | 约定 |
|----|------|
| 标准名 | 文本输入框 |
| 默认能力（不写） | 手工输入 |
| 说明列必写 | 是否多值（回车识别）、匹配方式（包含 / 精准）、默认值 |

**多编号：**

```markdown
|方案编号|文本输入框|1. 支持输入多个，通过回车识别每个编号。<br>2. 查询逻辑：输入内容包含方案编号字段内容的数据。|
```

**精准匹配：**

```markdown
|物品名称|文本输入框|1. 查询逻辑：物品名称字段与查询输入值精准匹配。|
```

---

### 复选框

| 项 | 约定 |
|----|------|
| 标准名 | 复选框 |
| 说明列必写 | 枚举项、默认勾选、互斥/可同时选、显示状态下必填、置灰条件 |

**范例（区间维度）：**

```markdown
**控件：**复选框；按区间配置时至少选 1 项；必填。
**枚举项（固定展示顺序）：**票-操作结算重量段、…
**限制：**仅操作类型为移动时子托三项可选；非移动时子托三项置灰。
```

## 2. 组织 / 权限 / 人员

### 数据权限组件

| 项 | 约定 |
|----|------|
| 标准名 | 数据权限组件 |
| 何时用 | 按**当前登录人数据权限**选分拨中心等 |
| 默认能力（不写） | 权限范围内可选、搜索选择 |
| 说明列必写 | 单选/多选、可输入内容范围、置灰与必填联动 |

**查询（多选）：**

```markdown
|分拨中心|数据权限组件|支持多选。|
```

**弹框（单选 + 联动）：**

```markdown
**控件：**数据权限组件，单选；**可输入内容：**分拨、服务站、集配站。**限制：**纬度为空或选择全国时，置灰不可填写。可输入状态为必填。
```

---

### 分拨中心组件

| 项 | 约定 |
|----|------|
| 标准名 | 分拨中心组件 |
| 何时用 | 选网点/分拨/省区等组织，**范围由本页写死**（不一定等于数据权限） |
| 与数据权限组件区别 | 数据权限组件跟登录人权限；分拨中心组件跟业务可选范围（如所有网点、仅分拨性质等） |
| 说明列必写 | 单选/多选、可输入范围、查询匹配方式（包含或交集） |

**查询：**

```markdown
|上一站|分拨中心组件|1. 支持输入多个。<br>2. 输入支持范围：所有网点。<br>3. 查询逻辑：上一站字段与查询输入值存在包含或交集关系的数据。|
```

**弹框组合（类型 + 组件 + 含下级）— 必须展开：**

```markdown
1. **上一站：**
    1. **控件1：**单选下拉框；**枚举项：**上一站网点、上一站分拨、上一站省区；**默认值：**上一站网点；
    2. **控件2：**分拨中心组件，支持多选；**可输入范围：**联动控件1…
    3. **控件3：**复选框；**限制：**仅当输入内容里包含一级网点时可选；否则置灰。
```

报表里若写「分拨中心组件（多选）」与上同义，维护页优先用「分拨中心组件」+ 说明里写多选。

---

### 人员组件 / 员工组件

| 项 | 约定 |
|----|------|
| 标准名 | 维护页、配置页优先 **人员组件**；报表可与现有文档对齐用 **员工组件**（同页内不要混用） |
| 默认能力（不写） | 按姓名/工号选人 |
| 说明列必写 | 单选/多选、可输入范围、默认值、查询逻辑；若受分拨条件约束要写明 |

```markdown
|创建人|人员组件|1. 支持多选。<br>2. 默认值：**空，该查询条件不生效、不参与数据过滤。**<br>3. 查询逻辑：选中内容包含创建人字段内容的数据。|
```

```markdown
|人员姓名|人员组件（多选）|1. 支持输入姓名、工号模糊匹配；受分拨中心条件约束。<br>2. 默认值：**空，该查询条件不生效、不参与数据过滤。**<br>3. 查询逻辑：选中人员与人员姓名、人事工号匹配的数据。|
```

（上例「模糊匹配」属于本页明确要求的匹配方式，可写；不要写「弹出人员选择器」这类常识。）

---

### 岗位组件

| 项 | 约定 |
|----|------|
| 标准名 | 岗位组件 |
| 说明列必写 | 多选、默认值、可输入范围（所有岗位 / 仅转运岗位） |

```markdown
|岗位|岗位组件|1. 支持多选。<br>2. 默认值：空。<br>3. 可输入范围：系统所有岗位（或：仅转运岗位）。|
```

## 3. 时间类

| 标准名 | 何时用 | 说明列常写 |
|--------|--------|------------|
| 时间日期选择器 | 弹框单个时点（含时分秒） | 默认值、必填、最小可选日、**时分秒固定拼接** |
| 区间时间选择器 | 查询起止（含时分秒） | 默认空是否过滤、与字段时间有交集 / 落在区间内 |
| 区间日期选择器 | 查询起止（到日） | 同上 |
| 区间时间日期选择器 | 报表常用起止（含时分秒、常有默认月） | 默认区间、最大跨度 |
| 区间月份选择器 | 按月统计 | 默认当月～当月、与统计月份有交集 |

名称在「区间时间 / 区间日期」上历史文档略有混用：**同一 PRD 内统一一种**；有时分秒用带「时间」的名字。

**查询：**

```markdown
|有效期限|区间时间选择器|1. 默认值：**空，该查询条件不生效、不参与数据过滤。**<br>2. 查询逻辑：开始至结束时间与选择的时间有交集的数据。|
```

**弹框：**

```markdown
**控件：**时间日期选择器；**默认值：**空；时分秒固定拼接：15:00:00；必填。
输入限制：当天 15 点前最小可选当日；当天 15 点及之后最小只能选次日。
```

## 4. 数字 / 比例 / 区间

### 百分比数字输入框

常与「+/-」下拉组成组合控件。

```markdown
**控件：**单选下拉框+百分比数字输入框组合
1. 单选下拉框：枚举项 -、+；默认空；必填；
2. 百分比数字输入框：最大三位整数+两位小数；默认空；必填；
```

### 区间式数字输入框

| 项 | 约定 |
|----|------|
| 标准名 | 区间式数字输入框 |
| 说明列必写 | 整数/小数位、前闭后开、段内不重复、增删行、无上界填法（如 99999.99） |

### 配置行表格（业务组合，非单一控件）

区间维度勾选后动态出列 + 多行上下限 + 调整比例时，按「配置行表格」整段写，不要拆成无数个无名输入框。参照货量重量配置 V1.1.0 调整比例写法。

## 5. 查询区布局（非控件，常附一句）

```markdown
|展示规则：查询条件默认展示 3 行。|||
```

放在查询条件表最后说明行，不占字段名。

## 6. 反例 → 正例

| 不推荐 | 推荐 |
|--------|------|
| 控件：下拉选择器，支持搜索多选，点击后弹出… | 控件：多选下拉框；说明里写枚举、默认、查询逻辑 |
| 组织树组件（目录无此名） | 先确认是「数据权限组件」还是「分拨中心组件」 |
| 把分拨中心组件的搜索、面包屑写进 PRD | 只写：多选、可输入范围、查询逻辑 |
| 组合上一站只写「分拨中心组件」 | 拆控件1/2/3：类型、组件、含一级网点 |

## 7. 维护与扩展

### 自动入库规则

用户描述了目录里没有的组件（正式名 + 用途）后，**同轮对话内**追加到本文件，并在 `SKILL.md`「组件选型速查」加一行。用户已说清楚时不必再问「要不要加进目录」。

仅本页一次性的特殊拼装：写在当页 PRD，**不**入库；用户明确说「以后都用这个名 / 加进目录」再入库。

已有 PRD 用了旧叫法：新文档跟目录走；改旧文档需用户明确要求，不擅自批量替换。

### 新条目模板（复制后填）

按所属大类插入（基础 / 组织人员 / 时间 / 数字区间等）；对不上就新建小节。

```markdown
### {标准名}

| 项 | 约定 |
|----|------|
| 标准名 | {与用户确认的正式叫法} |
| 何时用 | {一句话业务意图} |
| 默认能力（不写） | {大家熟悉、PRD 不必复述的交互} |
| 说明列必写 | {枚举/范围/单多选/默认/查询逻辑等本页常写项} |
| 易混 | {与哪个已有组件区分；无则删本行} |

**查询条件范例：**

\`\`\`markdown
|{字段名}|{标准名}|{只含本页差异的说明}|
\`\`\`

**弹框范例（可选）：**

\`\`\`markdown
**控件：**{标准名}；…
\`\`\`
```

同时在 `SKILL.md` 选型表追加：

`| {业务意图} | {标准名} |`
