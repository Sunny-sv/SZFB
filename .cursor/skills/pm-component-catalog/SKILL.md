# 通用组件描述规范

## 何时使用

- 写维护页 / 报表 PRD 的查询条件、弹框输入项、列表相关控件时
- 用户说「用现有组件」「按货量重量配置那种写法」「不要重新描述控件」时
- 与 `pc-maintain-prd`、`pc-report-prd`、`pc-prototype`、`pm-prd-writing-style` 同时使用：本 skill 管**控件怎么点名、说明列写什么**

## 核心原则

| 原则 | 要求 |
|------|------|
| 点名不重述 | 控件列写系统约定组件名；说明列**不写**该组件人人都懂的交互（怎么点、怎么搜、怎么多选） |
| 只写本页差异 | 说明列只写：枚举/数据源、默认值、空是否过滤、查询逻辑、单多选、可选范围、必填、置灰/联动、输入限制 |
| 名称统一 | 同一组件全项目用同一叫法，见 [reference.md](reference.md)；不要 invent 新名字 |
| 组合才展开 | 多控件拼成一组（如「类型下拉 + 分拨组件 + 含下级」）才分层写清各子控件；单组件不拆 |
| 缺则入库 | 目录没有的组件，用户描述清楚后**当场写入** reference + 选型表，不只口头用一次 |

## 写法：查询条件

表格列：`名称 | 控件 | 说明`

```markdown
|分拨中心|数据权限组件|支持多选。|
|创建人|人员组件|1. 支持多选。<br>2. 默认值：**空，该查询条件不生效、不参与数据过滤。**<br>3. 查询逻辑：选中内容包含创建人字段内容的数据。|
|状态|多选下拉框|1. 枚举项：生效中、待生效、已失效、终止。<br>2. 默认值：**生效中。**<br>3. 查询逻辑：选中内容包含状态字段内容的数据。|
```

**说明列必写（有则写）：**

1. 枚举项或数据来源（字典名 / 业务表说明，不确定标「待确认」）
2. 默认值（空是否参与过滤写清楚）
3. 查询逻辑（包含 / 有交集 / 精准匹配 / 时间有交集）
4. 单选还是多选、可选范围（若与组件默认不同）
5. 联动、置灰（若有）

**说明列不要写：**

- 「支持搜索」「支持模糊匹配」「点击弹出选择器」等通用交互
- 组件内部 UI 结构（除非本页有特殊改动）

## 写法：弹框 / 表单输入项

字段描述固定句式：

> **控件：**{组件名}；**枚举项/可输入范围：**…；**默认值：**…；**必填/非必填**；**限制：**…

```markdown
1. **分拨中心：**
    1. **控件：**数据权限组件，单选；**可输入内容：**分拨、服务站、集配站。**限制：**纬度为空或选全国时置灰。可输入状态为必填。
2. **自有调整、外包调整**
    1. **控件：**单选下拉框+百分比数字输入框组合
        1. 单选下拉框：**枚举项：**-、+；**默认值：**空；必填；
        2. 百分比数字输入框：**最大输入：**三位整数+两位小数；**默认值：**空；必填；
```

## 何时必须展开写全

以下情况**不能**只写组件名，要写清本页规则：

| 场景 | 要写什么 |
|------|----------|
| 组合控件 | 每个子控件的类型、枚举、默认、联动（如上一站：类型下拉 + 分拨组件 + 含一级网点） |
| 区间配置行 | 列怎么动态增删、上下限格式、前闭后开、增删行规则 |
| 与通用行为不同 | 如：人员组件本页只能单选、分拨组件可选范围不是「所有网点」 |
| 时分秒固定拼接 | 如开始时间固定 `15:00:00`、结束时间固定 `14:59:59` |
| 业务校验 | 重复校验、交集、必填在显示状态下才生效等 |

## 组件选型速查

写字段前先对号入座（详细约定见 [reference.md](reference.md)）：

| 业务意图 | 用这个名字 |
|----------|------------|
| 固定枚举，选一个 | 单选下拉框 |
| 固定枚举，选多个 | 多选下拉框 |
| 自由文本 / 多编号回车 | 文本输入框 |
| 按登录人数据权限选分拨等 | 数据权限组件 |
| 选网点/分拨/省区（不绑数据权限或范围另定） | 分拨中心组件 |
| 选系统人员 | 人员组件（报表里也可用「员工组件」，同义时统一一种） |
| 选岗位 | 岗位组件 |
| 起止日期或日期时间 | 区间时间选择器 / 区间日期选择器 / 时间日期选择器（按是否含时分秒） |
| 起止月份 | 区间月份选择器 |
| 勾选多项（非下拉） | 复选框 |
| 上下限数字段 | 区间式数字输入框 |
| +/- 百分比 | 单选下拉框+百分比数字输入框组合 |

目录里没有、或本页交互明显不同 → 走下方「新组件自动入库」，不要瞎编名字、也不要只用一次不落目录。

## 新组件自动入库（强制）

写 PRD / 原型时若控件不在 [reference.md](reference.md)：

1. **先对齐，再入库**  
   - 信息不够：问清 **正式名称**、**干什么用**、**说明列通常要写什么**、**默认能力哪些不用写**（可对照已有条目）。  
   - 用户已经讲清楚（含「就叫××组件」「和××类似但…」）：**不要再确认「要不要加目录」**，直接入库。

2. **当场改两个文件**（同一次回复内完成，与当前 PRD 一并交付）  
   - [reference.md](reference.md)：按同类章节追加条目（标准名 / 何时用 / 默认能力不写 / 说明列必写 / 1～2 个范例）。  
   - 本文件「组件选型速查」表：加一行「业务意图 → 标准名」。  
   - 若与旧名同义：在条目里写清「同××，本页统一用××」，避免两套叫法。

3. **回复里带一句**  
   - 例如：`已将「外包公司组件」写入组件目录。`  
   - 然后继续用新标准名写当前文档。

4. **不要做的**  
   - 不 invent 未确认的正式名。  
   - 不擅自批量改历史 PRD 里的旧叫法（除非用户明确要求）。  
   - 不把仅本页一次性的特殊拼装当成新通用组件（那种按「组合才展开」写在当页即可；若用户说「以后都这么叫、要进目录」再入库）。

入库条目模板见 [reference.md](reference.md)「维护与扩展」。

## 与其他 skill 的分工

| Skill | 管什么 |
|-------|--------|
| 本 skill | 控件叫什么、说明列写差异不写常识 |
| `pc-maintain-prd` / `pc-report-prd` | 章节结构、查询/按钮/列表模板 |
| `pm-prd-writing-style` | 只写有用改动、表格分点、通俗说法 |
| `pc-prototype` | 线框里放什么控件形态 |

## 产出前自检

- [ ] 控件列用的是目录里的标准名，没有「下拉选择器」「组织树」等临时叫法
- [ ] 说明列没有复述「可搜索、可多选弹层」等通用交互（除非本页改了）
- [ ] 枚举、默认值、空是否过滤、查询逻辑已写清
- [ ] 组合控件已拆子控件；单组件未过度展开
- [ ] 与货量重量配置等已有 PRD 叫法一致
- [ ] 若本轮用了目录外组件：已写入 reference + 选型表，并在回复中说明
