# 需求讨论：主动补方案与待确认

## 何时使用

- 讨论阶段：梳理需求、对齐方案、写讨论结论/计划
- 用户丢对接纪要、会议要点、口述需求，让「整理一下 / 出计划 / 列方案」
- 用户说「有没有更好的做法」「行业一般怎么做」「帮我挑挑刺」

**不替代：** 原型用 `pc-prototype`；PRD 用对应 PRD skill。本 skill 管**讨论与计划怎么想、怎么呈给用户确认**。

## 核心原则

| 原则 | 要求 |
|------|------|
| 对接是输入不是天花板 | 对接里聊到的是业务诉求与约束；计划**不得**只做「纪要翻译」 |
| 主动补视角 | 每次梳理至少带上：行业常见做法 / 自己推演的备选 / 易漏场景与风险（能补则补） |
| 建议 ≠ 已决策 | 补充内容一律标 **待确认** 或 **假设 A/B**；不写成「已定」「就按这个做」 |
| 短、可拍板 | 先给结论与选项对比，再展开；一次 3～5 个关键确认点，不堆 20 页 |

与 `pm-core` 一致：不替用户拍库存口径、计费、合规等业务结论；不编造公司内部规则。

## 输出结构（讨论 / 计划默认）

按这个骨架写，可按篇幅裁剪，但**「待你确认」不可省**：

1. **对接诉求摘要**（2～5 条）— 对方要什么、已知约束  
2. **建议方案方向** — 在对接基础上整理后的主路径（标明哪些来自对接、哪些是补充）  
3. **行业 / 常见做法提醒** — 物流/WMS/转运结算等同类问题通常怎么做；与本次差异点  
4. **备选方案** — 至少 1 个与主路径不同的选项；用表格写利弊、适用条件  
5. **遗漏与风险** — 角色、异常、权限、时效、历史数据、跨系统口径等对接没提但容易踩坑的  
6. **待你确认** — 编号列表，每项可答「要 / 不要 / 改成…」；确认前不进入细原型/长 PRD  

## 怎么「自行补充」（检查清单）

梳理时过一遍，有则写入「提醒 / 备选 / 待确认」，没有把握就标待确认，不装成事实：

| 检查项 | 要问自己的 |
|--------|------------|
| 角色与权限 | 谁维护、谁只读、分拨/全国/总部是否要隔离 |
| 时效与状态 | 生效/待生效/失效、能否终止、改历史怎么算 |
| 匹配与优先级 | 多规则命中谁赢、未命中怎么办（对接常漏） |
| 异常与边界 | 空值、重复、部分失败、导入整份失败、跨月/跨日切点 |
| 审计与追溯 | 要不要变更记录、对比、导出留痕 |
| 与上下游口径 | 维护页、计算、报表是否同一套字段/重量口径 |
| MVP 切分 | 能否先上统一规则、特例后置；对接要的是否一次做完过重 |
| 行业对照 | 计泡/结算重量、波次、库位、绩效提成等，同类产品常见能力是否值得提一句 |

## 写法约定

**来源标注（建议）：**

| 标记 | 含义 |
|------|------|
| 【对接】 | 会议/纪要/用户原话已明确 |
| 【行业】 | 行业或同类系统常见做法，供对照 |
| 【建议】 | AI 推演的备选或补强，须用户确认 |
| 【风险】 | 不处理可能出的问题 |

**选项对比表示例：**

| 方案 | 做法 | 好处 | 代价 / 风险 | 建议 |
|------|------|------|-------------|------|
| A【对接】 | … | … | … | 待确认 |
| B【建议】 | … | … | … | 待确认 |

**禁止：**

- 只把对接原文扩写成「计划」而无任何补充视角  
- 把【建议】【行业】写成已拍板结论  
- 用内部未确认的单号规则、对接系统名、真实客户数据充数  
- 讨论阶段直接开写完整 PRD 细表（除非用户明确只要 PRD 且声明讨论已完成）

## 与计划工具的关系

若同时用 Cursor 计划 / 待办：计划条目里把 **待确认项** 单列；未确认前，不把「按建议方案实现」写成已锁定任务。

## 产出前自检

- [ ] 不只是对接内容复述，有【行业】或【建议】或遗漏风险至少一类  
- [ ] 有「待你确认」编号列表，且可直接回答  
- [ ] 主路径与备选的来源已区分（对接 vs 补充）  
- [ ] 未把未确认项写成已决策  
- [ ] 篇幅可控：关键确认优先，细节留给确认后的原型/PRD  

## 短示例

**对接只说：**「要按分拨配一个重量调整比例。」

**不应只输出：** 做分拨维度配置页，字段=比例。

**应补充并待确认，例如：**

1. 【行业】同类常区分统一规则 vs 特例（线路/货主/重量段）；是否本期只要统一？  
2. 【建议】是否要全国默认 + 分拨覆盖，避免每仓都配一遍？  
3. 【风险】比例上调/下调、生效时间切点、与现有操作结算重量计算谁先谁后 — 对接未提。  
4. **待你确认：**（1）只要分拨统一？（2）要不要全国兜底？（3）要不要变更记录？
