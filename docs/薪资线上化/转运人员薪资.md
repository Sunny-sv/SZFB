# 二、要解决的问题

|**版本**|**问题**|
|---|---|
|V1\.8\.0<br>|1. 每月 8 号人事发出初版薪资后，若出现薪资核算错误，**出勤天数不准是最高频诱因**，但此时考勤表已完成签字并发起 OA 审批，现有流程不支持总部人员直接修改薪资核算对应的出勤天数数据。<br>2. 出勤天数修改后，无法对对应人员的薪资进行快速重算，且缺少修改前后的薪资差异记录，导致薪资纠错流程效率低、追溯难。|
|V1\.9\.0|新增的大票业务因货源结构与现有普货存在明显差异，对应的薪资计算规则无法与普货共用，需要进行差异化核算；但当前系统不支持大票业务薪资的独立计算，需人工拆分普货与大票的薪资数据，存在效率低、易出错的问题。|
|V2\.0\.0<br>|1. 特定时间段离职、复职的员工，存在反复离职复职现象，影响分拨作业效率。<br>2. 此类复职员工无差异化薪资管控，与正常入职员工薪资无区分，无法约束其流动，也影响在职员工公平感。<br>3. 复职员工春节前后返岗积极性不足，可能导致分拨人员短缺，影响节日业务开展。<br>4. 针对该类员工的复职管控方案不明确，流程、薪资、激励规则模糊，增加管理难度。|
|V2\.1\.0|皮带机操作模式的薪资目前依赖人工线下核算，未接入系统自动计算，存在效率低、易出错、数据不透明、核对成本高的问题。|
|V2\.2\.0|目前集笼功能仅支持直发货整笼发往目的分拨并完成卸车，**不支持经停货物整笼直发目的分拨**。郑州、睢宁线路中近三分之一为经停路由，现有集笼模式无法匹配实际中转业务；因此需新增专属集笼操作模式，同时该新模式需要独立核算薪资提成。|
|V2\.4\.0|1. 机叉叉车组长「特殊人员」改为维度=人员独立方案，算薪需同步匹配。<br>2. 绩效薪资需与百万件破损挂钩，按连续破损动态升降档。<br>3. 绩效薪资上下限改为**累计升降档**校验口径，越界时调减当月升/降档金额。<br>4. 人员工资明细新增「升降档金额」字段；评估窗口内岗位须一致才参与升降档。|

# 三、业务分析

## 3\.1  产品方案

|**版本**|**调研结果**|
|---|---|
|V1\.8\.0|改出勤情况较多，需要线下人工重算薪资然后再补差价，比较繁琐，希望系统可以自动重算。|
|V1\.9\.0|**系统支持大票货薪资的单独计算**，实现大票与普货薪资核算的分离处理。|
|V2\.0\.0|建立管控方案，通过差异化薪资约束流动，设置返岗奖励，规范复职管理。|
|V2\.1\.0|皮带机相关工作量需人工统计、手动算薪，流程繁琐且易产生误差，希望系统支持皮带机操作模式薪资自动计算。|
|V2\.2\.0|1. 新增独立的集笼单价配置项，针对经停货物整笼集笼模式设置专属计价标准。<br>2. 系统按照配置的对应单价，自动完成该类集笼作业的薪资提成计算。|

# 四、需求分析

## 4\.1 需求价值

|**版本**|**需求价值**|
|---|---|
|V1\.8\.0|1. **准确性价值**：修改出勤天数后可自动触发对应人员的薪资重算，减少人工手动重算的失误率，提升薪资核算结果的准确性。<br>2. **可追溯价值**：记录出勤天数修改动作及薪资差异数据，形成完整的修改台账，便于后续追溯薪资错误原因、核对薪资数据，同时满足合规审计的要求。|
|V1\.9\.0|1. 提升效率<br>    1. 替代人工拆分普货与大票薪资数据的操作，实现大票薪资的自动化独立核算，缩短薪资结算的整体周期；<br>2. 降低成本<br>    1. 降低人工核算的误差率，减少因薪资计算错误引发的员工反馈、薪资补发、纠纷协调等后续隐性成本；同时减少核算人员在重复数据拆分工作上的人力投入。<br>3. 合规性<br>    1. 实现大票与普货薪资核算规则的分离管理，两类业务的计算逻辑、数据来源清晰可追溯，同时精准匹配大票业务的作业量与薪资标准，保障薪资核算的公平性。<br>4. 新业务支撑<br>    1. 为大票新业务的顺利落地扫清系统能力障碍，避免因薪资核算环节的短板制约大票业务的市场拓展节奏；同时基于大票薪资的独立核算数据，可反向支撑大票业务的薪资规则迭代、人力成本建模等工作，推动新业务的可持续发展。|
|V2\.0\.0<br>|1. 运营稳定：约束人员反复流动，保障分拨作业连续性，提升作业效率。<br>2. 管理规范：明确复职薪资、激励规则，填补管理空白，降低管理成本。<br>3. 成本优化：通过差异化薪资控制用工成本，借助返岗奖励减少临时用工需求。<br>4. 人员激励：平衡在职员工公平感，提升复职员工返岗意愿及留存率。<br>5. 长效支撑：形成标准化管控机制，助力建立稳定人员队伍，支撑业务发展。|
|V2\.1\.0|1. **提升核算效率：**替代人工统计与计算，减少重复劳动，缩短薪资结算周期。<br>2. **保证数据准确：**消除人工计算、录入误差，避免薪资错发、补发及员工申诉。<br>3. **规范薪资流程：**统一皮带机薪资计算口径，实现数据可追溯、流程标准化。<br>4. **降低管理成本：**减少核算人力投入，提升分拨整体薪资管理精细化水平。|
|V2\.2\.0|1. 适配经停路由业务场景，补齐经停货物整笼集笼能力，覆盖郑州、睢宁等线路的中转作业需求。<br>2. 实现不同集笼模式薪资规则区分，计价逻辑与实际作业场景匹配，保障薪资核算公平合理。<br>3. 依靠系统自动计算提成，减少人工统计、核算工作量，提升薪资结算效率与数据准确性。<br>4. 完善集笼作业规则体系，实现直发、经停两类集笼业务分类管理，提升业务管控精细化水平。|

### V2\.4\.0

#### 本版改动概述

1. **绩效工资**：标准取报表「保底绩效」；薪资表本地乘绩效分、出勤系数得实发。

2. **质量绩效**：标准取报表「质量绩效」；薪资表本地乘绩效分、出勤系数得实发。

3. **绩效工资发放门槛**：对比项在基础提成 \+ 阶梯提成 \+ 协助提成基础上再加「大票提成」；未达门槛或无绩效分 → 对应实发为 0。

4. **上限调整**：对比项再加破损、大票提成、特殊补贴（特殊货物、特殊车线、特殊时段）、车线调整差异。

#### 方案匹配（叉车组长、机叉司机）

1. 凡需从【基础提成方案维护】机叉叉车组长绩效 tab 读取方案配置的字段（基本薪资、责任货量提成、基础提成、上限调整、移动提成上下限等），均按下列规则匹配**一条方案**：

    1. **先匹配维度 = 人员**：分拨中心 \+ 人事工号 \+ 岗位 \+ 算薪月份；

    2. **未命中 → 匹配维度 = 分拨**：分拨中心 \+ 岗位 \+ 算薪月份；

    3. 须同时满足：

        - **真实有效期限**包含算薪月份；

        - 状态**存在过「使用中」**；

        - 命中多条 → 按方案类型 **总部 \> 临时 \> 长期** 取一条。
