#### 质量绩效计算

1. **前提**：命中方案「是否关联破损」= 是；= 否时质量绩效 = 0，不执行下列步骤。

2. 质量绩效是**方案周期内的累计值**，不是只存当月这一档。先按规则得出当月**应给**的升/降档金额，再与上月累计叠加，最后按保底 \+ 质量绩效做上下限调减。标准值**不**乘绩效分、出勤系数。

3. **评估窗口 N**（第二步）只用于判当月是否升/降档；**累加范围**是同一方案周期内各月，不是只加窗口 N 个月。

4. **第一步：绩效月份自身条件**（须全部满足，否则**质量绩效** = 0，不执行后续步骤）

    1. 当月每一天岗位相同（未转岗）。

    2. 当月每一天算薪分拨相同（未转分拨）。例：绩效月份为 6 月，6 月 15 日前在 A 分拨、15 日后在 B 分拨 → 不满足。

    3. 当月每一天在职状态均为在职。

    4. 非新入职月：入职日期所在月份 ≠ 绩效月份。

5. **第二步：评估窗口与有效月**

    1. 连续评估月数 **N**：取命中方案「连续评估月数」字段值。

    2. **评估窗口**：绩效月份之前的 **N** 个自然月，**不含绩效月份本身**。例：绩效月份为 6 月、N=2，窗口为 4 月、5 月。

    3. **有效月**：评估窗口内的每个自然月，须**同时满足**下列 4 条；任一不满足则该月**不算有效月**：

        1. 岗位与绩效月份一致，且该月每一天岗位相同（未转岗）。

        2. 算薪分拨与绩效月份一致，且该月每一天算薪分拨相同（未转分拨）。

        3. 该月每一天在职状态均为在职。

        4. 非该月新入职月：入职日期所在月份 ≠ 该自然月。

    4. 有效月数不足 **N** → 无法满足「连续 N 个月」→ **质量绩效** = 0，不执行后续步骤。

6. **第三步：读方案参数**

    1. 升降档判定方式（满足其一 / 同时满足）、升降档绩效分门槛、升档金额、降档金额、绩效薪资及上下限、质量绩效累计方式（重新计算 / 沿用上月）等，均取命中方案。

7. **第四步：绩效分**

    1. 取【绩效分导入】对应岗位 tab；**按评估窗口内每个有效月**分别取该月绩效分。某一有效月没有绩效分 → **绩效分相关条件视为不满足**（破损相关条件仍可单独成立）。

8. **第五步：判升档、判降档**

    1. 须绩效月份自身条件满足，且有效月数 = N。先分别判断升档是否成立、降档是否成立，再按「冲突处理」取金额。

    2. **升降档判定方式 = 满足其一**

    |    **方向**|    **条件**|
    |---|---|
    |    升档|    **满足其中一个即可**：① 连续 N 个有效月都没有破损；② 连续 N 个有效月绩效分都不低于升降档绩效分门槛|
    |    降档|    **满足其中一个即可**：① 连续 N 个有效月都有破损；② 连续 N 个有效月绩效分都低于升降档绩效分门槛|

    3. **升降档判定方式 = 同时满足**

    |    **方向**|    **条件**|
    |---|---|
    |    升档|    连续 N 个有效月都没有破损 **并且** 连续 N 个有效月绩效分都不低于升降档绩效分门槛|
    |    降档|    连续 N 个有效月都有破损 **并且** 连续 N 个有效月绩效分都低于升降档绩效分门槛|

    4. **冲突处理**

    |    **升档成立**|    **降档成立**|    **当月应给升/降档**|
    |---|---|---|
    |    是|    是|    0（例：连续 2 个月绩效分都高于门槛，且连续 2 个月都有破损）|
    |    是|    否|    方案升档金额（正数）|
    |    否|    是|    方案降档金额（负数）|
    |    否|    否|    0|

9. **第六步：方案周期累加与上下限调减**

    1. 取**上月累计**（见下一节「上月累计取值」）。

    2. **未调减质量绩效** = 上月累计 \+ 当月应给升/降档。

        - 当月再升档：在上月累计上再加一档。例：7 月质量绩效已是 500，8 月再升档 500 → 未调减 = 1000。

        - 当月不升不降（应给 = 0）：未调减 = 上月累计（沿用）。

        - 当月降档：在上月累计上再减一档（加负的降档金额）。

    3. 用保底绩效（方案「绩效薪资」）、绩效薪资上限、绩效薪资下限做调减：

        - 保底 \+ 未调减质量绩效 **高于** 上限 → 质量绩效 = 上限 − 保底。

        - 保底 \+ 未调减质量绩效 **低于** 下限 → 质量绩效 = 下限 − 保底。

        - 否则 → 质量绩效 = 未调减质量绩效。

    4. **举例**（升档金额 500，保底 2500，上限 4000）：

    |    **绩效月份**|    **当月应给**|    **上月累计**|    **未调减**|    **质量绩效（列表展示）**|
    |---|---|---|---|---|
    |    7 月|    \+500（首次升档）|    0|    500|    **500**|
    |    8 月|    \+500（再升档）|    500|    1000|    **1000**|
    |    9 月|    0（不升不降）|    1000|    1000|    **1000**|
    |    某月再累加会超上限|    \+500|    1400|    1900|    **1500**（上限 4000 − 保底 2500）|

10. **上月累计取值**

    1. 关联键：分拨中心 \+ 人事工号 \+ 岗位 \+ 绩效月份（取**上一自然月**）。

    |    **情况**|    **上月累计**|
    |---|---|
    |    本报表有上月数据，上月「方案编号」与当月命中方案**一致**，且未触发下方「人员中途变动」清零|    取上月「质量绩效」|
    |    无上月数据（首月上线、缺月等）|    0|
    |    上月方案编号与当月**不一致**（方案变更），且当月方案「质量绩效累计方式」= **重新计算**|    0（旧累计清掉）|
    |    上月方案编号与当月**不一致**（方案变更），且当月方案「质量绩效累计方式」= **沿用上月**|    取上月「质量绩效」（**不要求**方案编号一致）；取不到按 0|
    |    人员中途变动导致累计断开（见「其他情形」）|    0（变动前已涨金额不带到后续月；**不论**质量绩效累计方式选什么）|

11. **方案变更时怎么起算**（当月命中方案编号与上月不一致时，按当月方案「质量绩效累计方式」分支）

    1. **重新计算**

        1. 上月累计按 0；用新方案参数判升/降档与上下限；仍须有效月数 = N。
        2. 新方案刚生效时，评估窗口往往凑不齐 → 质量绩效为 0。新方案生效后的前若干月通常算不出质量绩效，须重新凑满 N 个有效月。
        3. **举例**（N=2）：7 月起用新方案 → 7 月、8 月通常质量绩效为 0，约 **9 月**起窗口才可能凑满 N 个有效月，才可能再出质量绩效。

    2. **沿用上月**

        1. 上月累计取本报表上一绩效月「质量绩效」（取不到按 0）；**不要求**上月与当月方案编号一致。
        2. **变更当月**即可按评估窗口判升/降档；升/降档金额、绩效分门槛、判定方式、绩效薪资上下限等全部按当月命中的新方案；评估窗口内绩效分、破损按各历史月实际取数。
        3. **举例**（N=2）：7 月改方案且沿用 → 7 月看 5 月、6 月的绩效分与破损。再例：8 月质量绩效已到 1000，9 月只改单价并选沿用上月 → 9 月上月累计仍为 1000，再按新方案判当月升/降档与上下限。

> **上线补跑**：正式上线前，**部分分拨**需一次性补跑最近若干月份历史标准，使首月算薪即可取到上月累计；未补跑分拨上月累计按 0。具体哪些分拨、补跑几个月，实施时按业务名单执行。

#### 其他情形

|**场景**|**规则**|
|---|---|
|人员中途变动（转岗 / 转分拨 / 离职等，方案未换）|1. **累计清 0**：变动月及之后不沿用变动前已涨的质量绩效。例：6 月质量绩效已到 1000，7 月转岗 → 不把 1000 带到后续月。<br>2. **重头开始**：按第一步、第二步重新凑有效月；有效月数不足 N → 质量绩效 = 0。<br>3. **起算节奏举例**（N=2）：7 月变动 → 7 月自身条件不满足 = 0；8、9 月窗口仍含无效月 → 仍为 0；约 **10 月**起窗口才可能凑满 N 个有效月，才可能再出质量绩效。<br>4. 与方案「质量绩效累计方式」无关，一律清零。|
|方案变更 + **重新计算**|上月累计按 0，重新凑有效月；新方案生效后前若干月质量绩效通常为 0，约凑满 N 个有效月后才可能出质量绩效（例 N=2：7 月新方案 → 约 9 月）。|
|方案变更 + **沿用上月**|取上月质量绩效（取不到按 0）；变更当月即可判升/降档；参数用新方案（例 N=2：7 月改方案 → 7 月看 5、6 月）。|
|绩效月份同月离职再入职|不满足绩效月份自身条件 → 质量绩效 = 0；并按「人员中途变动」清零累计|
|破损结算滞后|按完结时间计入，不对冲补发|
|窗口月转岗、转分拨、在职不满足|该月不算有效月；有效月数不足 N → 当月质量绩效 = 0|
