# 二、要解决的问题

|**版本**|**问题**|
|---|---|
|V4\.1\.0<br>|1. 目前需要根据兼职的货量类型设置不同的本职货量。<br>    1. 例如：装卸工兼职移动，本职货量需达到5吨；装卸工兼职流水线卸货，本职货量需达到6吨；|
|V5\.0\.0|新增的大票业务因货源结构与现有普货存在明显差异，对应的薪资计算规则无法与普货共用，需要进行差异化核算；但当前系统不支持大票业务薪资的独立计算，需人工拆分普货与大票的薪资数据，存在效率低、易出错的问题。|
|V5\.1\.0|目前空笼货量需要**在线下人工计算**，算完后再手动导入薪资表，全程依赖人工操作，流程繁琐，还容易出现计算错误、录入失误等问题。|
|V6\.0\.0|1. 特定时间段离职、复职的员工，存在反复离职复职现象，影响分拨作业效率。<br>2. 此类复职员工无差异化薪资管控，与正常入职员工薪资无区分，无法约束其流动，也影响在职员工公平感。<br>3. 复职员工春节前后返岗积极性不足，可能导致分拨人员短缺，影响节日业务开展。<br>4. 针对该类员工的复职管控方案不明确，流程、薪资、激励规则模糊，增加管理难度。|
|V7\.0\.0|当前仅支持**单个分拨、单条方案手动调整**，每月需重复配置，操作繁琐、效率低，易出现遗漏或配置错误。|
|V8\.0\.0|皮带机操作模式的薪资目前依赖人工线下核算，未接入系统自动计算，存在效率低、易出错、数据不透明、核对成本高的问题。|
|V8\.1\.0|目前集笼功能仅支持直发货整笼发往目的分拨并完成卸车，**不支持经停货物整笼直发目的分拨**。郑州、睢宁线路中近三分之一为经停路由，现有集笼模式无法匹配实际中转业务；因此需新增专属集笼操作模式，同时该新模式需要独立核算薪资提成。|
|V8\.2\.0|基础提成方案维护支持维度=人员后，交接单移动 tab 取叉车组长/机叉司机单价时需先匹配人员维度方案。|

# 三、产品方案

## 3\.1  产品方案

|**版本**|**调研结果**|
|---|---|
|V4\.1\.0|希望可以根据兼职的货量类型设置不同的本职货量。|
|V5\.0\.0|**系统支持大票货薪资的单独计算**，实现大票与普货薪资核算的分离处理。|
|V5\.1\.0|**由系统自动计算空笼货量**，替代线下人工核算和手动导入。|
|V6\.0\.0|建立管控方案，通过差异化薪资约束流动，设置返岗奖励，规范复职管理。|
|V7\.0\.0|每月重复配置方案工作量大，希望系统支持**提前维护方案 \+ 设置生效有效期**，到期自动切换，无需人工每月调整。|
|V8\.0\.0|皮带机相关工作量需人工统计、手动算薪，流程繁琐且易产生误差，希望系统支持皮带机操作模式薪资自动计算。|
|V8\.1\.0<br>|1. 新增独立的集笼单价配置项，针对经停货物整笼集笼模式设置专属计价标准。<br>2. 系统按照配置的对应单价，自动完成该类集笼作业的薪资提成计算。|
|V8\.2\.0|1. 岗位：机叉司机、叉车组长<br>    1. 三处联动改：<br>        1. 方案维护：支持维度=分拨/人员；取消特殊人员；可配破损关联升降档<br>        2. 交接单人员提成：叉车组长/机叉司机取价：先人员方案、再分拨方案（含支援分支）<br>        3. 转运人员薪资：绩效工资按破损动态升降；新增升降档金额|

|V7\.0\.0<br>|1. **提效减负：**取消重复人工配置，大幅减少月度操作工作量，提升方案维护效率。<br>2. **准确稳定：**避免人为漏配、错配导致的计算异常，保证规则执行统一准确。<br>3. **管理规范：**实现方案可预配置、可追溯、自动生效，满足多分拨规模化管理需求。<br>4. **灵活适配：**支持按时间段自动切换方案，适配旺季、淡季、政策变动等场景。|
