# 二、要解决的问题

|**版本**|**问题**|
|---|---|
|V1\.0\.0|总部制定新合托考核规则，需系统支持。|
|V2\.0\.0|分拨人员发现某些已卸车完成的交接单奖罚不合理\(过高\)时，需要重新调整目标值，但是目前无法调整。|
|V3\.0\.0<br>|目前最高纬度只到交接单，缺乏针对分拨整体合托情况统计以及同比等，对历史数据的变化展示。<br>当前分拨达成率的数据统计，**最高维度仅支持到交接单层级**，既缺乏对**分拨整体合托情况**的汇总统计能力，也未提供同比等历史数据变化的展示功能，无法直观反映分拨合托业务在不同周期内的趋势变化，制约了对分拨合托整体运营状况的研判。|
|V3\.1\.0<br>|1. 电商件重量轻、件数多，需与普货分开设置独立合托目标值；目前**合托规则维护已支持区分电商件与非电商件**，但**合托数据展示页面不支持按电商件 / 非电商件单独筛选查询**，无法分类查看、统计与核对，不便于运营管理与数据分析。<br>2. 业务部门因方案修改、规则调整等场景需**重刷核算数据**，但系统无面向业务用户的重刷功能，所有重刷操作必须由 IT 部门处理，流程繁琐、响应周期长，影响业务效率与数据更新及时性。|
|V3\.2\.0|回单货物均由文员统一扫描处理，不属于实际卸车作业范围，但当前系统在统计卸车员工合托考核时，**未剔除回单货量与回单托数**，造成合托考核数据失真，考核结果不准确、有失公平。|
|V4\.0\.0|1. 电商件越来越多，且不好合托（轻、件多、形态与普货差异大），运营需要在合托达成率里**直接看到电商、非电商各自的重量、托数、奖罚等**，便于分析和核对；现网只能看混在一起的总数，或按「方案产品类型」筛，**无法同屏对比两类货量**。<br>2. V3\.1\.0「是否电商件」按**方案产品类型**判定，与业务现行口径不一致；业务已约定按子单**订单来源**匹配系统参数认电商，报表侧需改口径并支持按**子托归属**做汇总拆分。|

# 三、业务分析

## 3\.1  用户调研

|**版本**|**调研结果**|
|---|---|
|V1\.0\.0|能直观看到每个交接单、人员的合托情况。|
|V2\.0\.0|系统可以支持修改已完成的交接单目标值。|
|V3\.0\.0|结合用户调研结果，相关业务人员的核心诉求为：一是实现**分拨维度的合托数据统计**，二是支持**分拨合托数据与历史数据的对比展示**。|
|V3\.1\.0|1. 合托规则已区分电商件与非电商件，希望**数据展示页面支持单独筛选查询电商件或非电商件**，实现分类数据查看与核对。<br>2. 方案调整后数据重刷需求频繁，依赖 IT 处理效率低、等待时间长，希望系统提供**业务人员可自主操作的数据重刷功能**。|
|V3\.2\.0|运营及考核管理人员提出诉求，希望系统在核算卸车员工合托考核指标时，**自动剔除回单对应的货量和托数**，只统计真实作业业务数据。|
|V4\.0\.0|运营希望合托达成率在分拨、交接单、人员等维度上，**前面看汇总、后面能区分电商/非电商**；电商怎么认，按子单订单来源与系统参数对齐，不再看方案产品类型。|

## 3\.2 产品方案

|**版本**|**产品方案**|
|---|---|
|V4\.0\.0|1. **判定口径**：子单「订单来源」与系统参数 `ELECTRIC_EC_ID` 列表精准匹配 →「是否电商件 = 是」；否则（含空）→ 否。汇总拆分不直接按子单是否电商件，而按「子托归属电商类型」（同托内按件数占比归属整托，件数相同归电商）。<br>2. **展示形态**：分拨汇总、交接单汇总、人员汇总、交接单人员明细采用「汇总列组 + 区分电商列组」；区分电商下列出「子托归属电商类型」，每个统计对象固定两行（电商、非电商）。子单明细不拆宽表，只新增「是否电商件」「子托归属电商类型」两列，并下线「是否电商件方案」。<br>3. **查询与导出**：各 Tab（除子单明细外）删除「是否电商件」筛选项，统一用「子托归属电商类型」过滤；新增「显示电商拆分」开关（默认勾选）。页面仍两行展示；导出走「一行扁列」——同一对象的汇总与电商/非电商指标并到一行（勾选拆分时含扁并列，取消勾选只导汇总）。<br>4. **历史与重刷**：电商相关字段按落库时参数快照保存；改参数不自动改已落库历史，需重刷才按新参数重算。本版只做报表统计分析，不涉及合托规则维护，也不改实际合托计算与奖罚生成规则。|

# 四、需求分析

## 4\.1 需求价值

|**版本**|**需求价值**|
|---|---|
|V1\.0\.0|降低合托数量，减少移动成本。|
|V2\.0\.0|使合托奖罚考核更公平|
|V3\.0\.0|1. **用户价值**<br>    1. 打破交接单层级的统计局限，让运营人员直接获取分拨整体合托数据，无需手动汇总、整合多份交接单数据；同时通过历史数据对比功能，直观掌握分拨合托业务的周期变化趋势，大幅降低数据整理与趋势分析的工作量，提升对分拨合托运营情况的研判效率。<br>2. **产品价值**<br>    1. 丰富产品的数据统计维度，完成从 “交接单明细” 到 “分拨层级汇总” 的能力升级；补充历史同比的数据对比功能，完善产品的数据分析与可视化能力。|
|V3\.1\.0|1. **区分电商件**<br>    1. **查询便捷：**支持按货物类型筛选，可独立查看电商件、非电商件合托数据，提升数据检索效率。<br>    2. **管理精细：**实现两类货物合托数据分类展示、清晰统计，贴合实际运营管理场景。<br>    3. **核对高效：**避免数据混杂，减少人工区分成本，提升数据核对与分析准确性。<br>    4. **展示规范：**统一数据展示口径，满足分类统计、报表查看的实际业务需求。<br>2. **重刷功能**<br>    1. **效率提升：**业务自主完成数据重刷，无需提报 IT、排队等待，大幅缩短数据更新周期。<br>    2. **降低 IT 负担：**减少 IT 大量重复性数据重刷运维工作，聚焦核心技术问题。<br>    3. **响应灵活：**快速适配方案变更，保障薪资 / 货量等数据及时、准确刷新。<br>    4. **流程自主：**赋予业务自主操作权限，提升运营管理灵活性与闭环能力。|
|V3\.2\.0|**考核公平公正：**剥离文员负责的回单数据，真实还原卸车实际合托作业量，避免无效数据拉低或干扰考核成绩。<br>**统计口径规范：**明确考核数据统计范围，区分回单业务与正常卸车业务，让合托考核规则更贴合实际岗位分工。|
|V4\.0\.0|1. **用户价值**<br>    1. 同一屏可对比汇总与电商/非电商货量、托数、奖罚，减少导出后手工拆分。<br>    2. 电商判定与业务参数口径一致，混托按子托归属规则可解释、可核对。<br>    3. 可用开关收起拆分列，兼顾日常只看汇总的习惯。<br>2. **产品价值**<br>    1. 合托达成率从「只能筛」升级为「可拆分对比」的分析能力。<br>    2. 统一子单判定与汇总归类两套字段，避免方案产品类型与订单来源两套口径并存。|

### V4\.0\.0

本文仅描述 V4\.0\.0 相对上一版的变更；未列出的内容默认未改。

#### 本版改动概述

1. **新增**电商判定与子托归属两套字段及公共计算规则（按订单来源参数；混托按件数占比归属整托）。
2. **调整**分拨汇总、交接单汇总、人员汇总、交接单人员明细列表：增加「区分电商」列组，每个对象固定电商/非电商两行。
3. **调整**子单明细：新增「是否电商件」「子托归属电商类型」；**删除**「是否电商件方案」。
4. **调整**查询条件：除子单明细外**删除**「是否电商件」筛选项；**新增**「子托归属电商类型」「显示电商拆分」（默认勾选）。
5. **调整**导出：分拨汇总、交接单汇总、人员汇总、交接单人员明细改为「一行扁列」（同一对象一行，汇总 + 电商列 + 非电商列并排；勾选拆分才含电商/非电商列）；子单明细导出不改。页面展示仍两行。
6. **调整**数据重刷：重刷需重算并写出电商相关字段及据此汇总的拆分结果。
7. 本版只做报表统计分析（展示拆分、查询、导出、重刷写出拆分结果）；不涉及合托规则维护，也不改实际合托计算与奖罚生成规则。

#### 公共规则

##### 1\. 是否电商件

子单「订单来源」与系统参数 `ELECTRIC_EC_ID` 列表**精准匹配** → **是**；否则（含空）→ **否**。  
用于子单明细展示，并参与计算「子托归属电商类型」。

##### 2\. 子托归属电商类型

同一子托内，按「是否电商件」分别统计件数：哪边件数多归哪边；**一样多归电商**。同托下各子单该字段相同。  
分拨 / 交接单 / 人员等汇总拆分时，**按本字段归类**，不按「是否电商件」。

> **举例**：托内电商 30 件、非电商 10 件 → 整托归电商。其中某单「是否电商件 = 否」，汇总拆分时重量仍进电商侧。

##### 3\. 参数与历史

落库时按**当时**参数算出并保存。只改参数、不重刷 → 历史不变；要按新参数重算 → 走数据重刷。
