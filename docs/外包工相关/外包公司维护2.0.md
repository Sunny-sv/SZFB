#### 外包公司类型同步

同一合作分拨、同一外包公司下，各 Tab 方案的「外包公司类型」应保持一致。保存成功后，按下列规则自动同步。

**一、触发条件**

|场景|是否同步|说明|
|---|---|---|
|编辑保存，且外包公司类型相对打开弹框时有变化|是|含当前方案状态为已失效的情况|
|编辑保存，外包公司类型未变化|否|—|
|新增保存成功|是|以本条保存的类型为准，对齐同主体其他方案|

**二、同步范围**

1. **找哪些方案**：先看本次保存填的「合作分拨」「外包公司」，再找这两个字段都一样的其他方案（两个都要相同，只同公司或只同分拨不算）。
2. **哪些 Tab**：装卸、移动、流水线卸货、流水线码货、流水线移动、流水线直装，全部找一遍。
3. **改哪些状态**：上一步找到的方案里，只改状态为生效中、待生效的，把它们的外包公司类型改成与本次保存一致。
4. **不改哪些**：已失效方案；本页已删除、查询不到的方案。

**三、已失效方案怎么处理**

1. **作为被同步方**：其他方案（含新增、编辑生效中/待生效）保存并触发同步时，**不会**改已失效方案的外包公司类型，避免把历史类型盖掉。
2. **作为编辑方**：已失效方案可以点开编辑并修改外包公司类型；保存时若类型有变化，仍按第二节规则，把同合作分拨、同外包公司下其他 Tab 的生效中、待生效方案同步成新类型；其他已失效方案仍不更新。
3. 小结：已失效「不被带着改」，但「自己改了可以带着改别人（仅生效中/待生效）」。

**四、提示与留痕**

1. 同步条数 N：统计本次被更新的其他方案条数，不含当前正在保存的本条。
2. 保存成功提示须说明同步结果与范围，例如：`保存成功。已同步更新 N 条其他方案的外包公司类型（不含本条；范围：同合作分拨+同外包公司、六 Tab、仅生效中/待生效，已失效不同步）`。若 N 为 0，提示无其他需同步的生效中/待生效方案（本条仍保存成功）。
3. 外包公司类型变更记入变更记录。

> **举例**：金华分拨 + 某外包公司，装卸有已失效 1 条、生效中 1 条，移动有生效中 1 条。在装卸已失效方案上将类型由临时工改为外请车并保存 → 本条更新为外请车；同步更新 2 条（装卸生效中、移动生效中）；其余已失效方案类型不变；提示 N=2。

#### 历史数据

上线时，本维护页仍能查到的方案，「外包公司类型」统一为 **临时工**（含生效中、待生效、已失效）。
