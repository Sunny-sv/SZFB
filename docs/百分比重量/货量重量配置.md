# 二、要解决的问题

|**版本**|**问题**|
|---|---|
|V1\.0\.0|当前操作结算重量计算完成后，缺少灵活的百分比配置减免/增加机制。运营成本部无法根据业务策略对操作结算重量进行差异化调整，所有货量统一按原始计泡计算，缺乏弹性运营能力。|
|V1\.1\.0|V1\.0\.0 特殊维护「调整比例」将计算类型（按票/按件/按子托）与区间类型（重量段/体积段）绑定，无法表达跨粒度组合条件。例如：无法配置「运单操作结算重量 ≥800kg 且 运单总件数 ≥50 件」才套用单独调整比例；亦无法在同一方案内同时约束票级重量与子单级重量。运营特例规则配置能力不足。|

# 三、产品分析

## 3\.1  产品方案

|**版本**|产品方案|
|---|---|
|V1\.0\.0|基于运营成本部实际业务场景：在操作结算重量计算完成后的链路中，增加一层可配置的百分比调整引擎，通过多维度规则匹配的方式，实现对货量重量的灵活上调或减免。|
|V1\.1\.0|在特殊维护 Tab 将调整比例升级为区间维度矩阵：支持票/子托/件 × 重量/体积/件数共 9 项维度任意多选组合；同一配置行内多维度 AND 命中后取该行调整比例。允许间断区间（仅维护关心档位），未覆盖货量不命中本方案、继续走方案层级兜底。统一维护 Tab 本期仍仅「不区分」，不做区间配置。|

## 4\.1 需求价值

|**版本**|**需求价值**|
|---|---|
|V1\.0\.0|1. 提升结算灵活性：支持按线路、产品类型等多维度对操作结算重量进行差异化调整，满足运营策略需要。<br>2. 减少人工干预：规则引擎自动匹配，无需人工调整<br>3. 策略可追溯：变更记录留痕，所有调整规则有据可查<br>4. 分拨中心自治：各分拨中心独立维护规则，互不干扰|
