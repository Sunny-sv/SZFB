# 二、要解决的问题

|**版本**|**问题**|
|---|---|
|V1\.0\.0|目前，用户在操作结算重量计泡方面遇到的挑战是系统无法支持按照特定的分拨中心、具体的操作环节，甚至细化到寄件人的层级来进行维护。这导致了在实际操作中缺乏灵活性，无法针对不同的情况进行精准的成本控制。|
|V2\.0\.0|当前操作结算重量计算完成后，缺少灵活的百分比配置减免/增加机制。运营成本部无法根据业务策略对操作结算重量进行差异化调整，所有货量统一按原始计泡计算，缺乏弹性运营能力。|
|V2\.1\.0|特殊维护「调整比例」将计算类型（按票/按件/按子托）与区间类型（重量段/体积段）绑定，无法表达跨粒度组合条件。例如：无法配置「运单操作结算重量 ≥800kg 且 运单总件数 ≥50 件」才套用单独调整比例；亦无法在同一方案内同时约束票级重量与子单级重量。运营特例规则配置能力不足。|

# 三、业务分析

## 3\.1  用户调研

|**版本**|**调研结果**|
|---|---|
|V1\.0\.0|用户希望系统能够提供更加灵活和精细化的管理方式，即允许根据不同的分拨中心、操作环节，乃至寄件人来分别设置和调整操作结算重量计泡规则。这种需求的根本目的在于实现更精确的成本管理和优化，以适应不同场景下的业务需求，最终达到降低成本的目标。|
|V2\.0\.0|基于运营成本部实际业务场景：在操作结算重量计算完成后的链路中，增加一层可配置的百分比调整引擎，通过多维度规则匹配的方式，实现对货量重量的灵活上调或减免。|
|V2\.1\.0|在特殊维护 Tab 将调整比例升级为区间维度矩阵：支持票/子托/件 × 重量/体积/件数共 9 项维度任意多选组合；同一配置行内多维度 AND 命中后取该行调整比例。允许间断区间（仅维护关心档位），未覆盖货量不命中本方案、继续走方案层级兜底。统一维护 Tab 本期仍仅「不区分」，不做区间配置。|

#### 本版改动概述

|序号|改动概述|
|---|---|
|1|**删除「计算类型」**：不再按票/按件/按子托决定取哪个粒度的重量或体积。|
|2|**区间匹配改版**：特殊维护「按区间配置」时，改为维护页勾选区间维度 \+ 配置行；**同一配置行内多维度须同时满足（AND）**。|
|3|**新增件数维度**：新增票\-件数段、子托\-件数段；件粒度仍无件数段。|
|4|**匹配条件数量调整**：计入项删除计算类型、笼统公斤段/体积段；新增「每启用 1 种区间维度 \+1」。|
|5|**配置行命中规则整段重写**（本版核心）。|

#### 区间维度与扫描取值

方案为「按区间配置」时，按维护页勾选的维度取扫描实际值：

|**区间维度**|**扫描时取什么值**|
|---|---|
|票\-操作结算重量段|运单操作结算重量|
|票\-体积段|运单体积|
|票\-件数段|运单总件数|
|子托\-操作结算重量段|子托开单预测操作结算重量–––|
|子托\-体积段|子托体积|
|子托\-件数段|子托件数（该子托下子单合计）|
|件\-操作结算重量段|子单操作结算重量|
|件\-体积段|子单体积|

#### 配置行命中规则

##### 区间模式 = 不区分

直接取方案上的轻实重调整、自有调整、外包调整，再按岗位取对应比例（外包工取外包调整，其他岗位取自有调整），按原百分比重量公式计算。

##### 区间模式 = 按区间配置

###### 第一步：取维度和扫描实际值

读维护页勾选的区间维度，取对应扫描实际值。

###### 第二步：检查能否匹配本方案

1. 已勾选的**每一个**维度，扫描时都必须能取到实际值。

2. 任一维度取不到数（为空、无效、系统算不出）→ **放弃本方案**，去下一层继续找。

3. **禁止**：某个维度没取到数就跳过该维度、只核对其余项，或当成「这一项不限制」仍判定命中。

> **举例**：方案勾选了「票\-操作结算重量段 \+ 票\-件数段」。扫描时运单操作结算重量能取到，但运单总件数取不到 → 本方案**直接不适用**。
> 
> 

###### 第三步：在配置行中找命中行

1. 方案下可维护多行配置；每行对已勾选维度分别维护下限～上限。

2. **同一行内须全部满足（AND）**：每个维度的实际值都须落在该行对应区间内，该行才算命中。

3. 区间判定：下限 ≤ 实际值 \< 上限。

###### 第四步：按命中结果处理

|**命中情况**|**处理方式**|
|---|---|
|命中 **1 行**|取该行的轻实重/自有/外包调整 → 按岗位取对应比例 → 算百分比重量|
|**0 行**命中|本方案区间未覆盖本批货（间断区间属正常）→ **放弃本方案**，继续下一层|
|**多行**同时命中|配置侧应已禁止行间重叠；若运行时仍出现 → 取「各启用维度上限加总最小」的行|

**整链路兜底**：特殊维护\-分拨、特殊维护\-全国、统一维护\-分拨、统一维护\-全国四层均无有效命中 → 百分比重量 = 子单操作结算重量（无调整）。

###### 业务示例

**配置（特殊维护 · 按区间配置）**

|**票\-操作结算重量段**|**票\-件数段**|**自有调整**|**外包调整**|
|---|---|---|---|
|800～99999\.99|50～999999|\-3%|\-5%|

**命中**：运单操作结算重量 = 900kg，运单总件数 = 60 件。

1. 900 落在 800～99999\.99 内，60 落在 50～999999 内 → 均满足 → 命中该行；

2. 扫描人为自有岗位 → 取 \-3%；

3. 按该行轻实重调整及原百分比重量公式计算。

**未命中**：运单操作结算重量 = 900kg，运单总件数 = 30 件 → 重量满足、件数不满足 → 本方案未命中 → 继续下一层；四层均无 → 百分比重量 = 操作结算重量。
